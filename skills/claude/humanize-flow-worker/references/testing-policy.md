# Testing Policy

Run tests that are proportional to the change.

## Choose tests from

- the approved plan,
- package scripts,
- existing test files near the changed code,
- project instructions in `AGENTS.md`, `CLAUDE.md`, or README.

## Minimum expectation

For implementation changes:

- run targeted tests when available,
- run lint/type checks when relevant,
- explain any tests you did not run.

## Evidence

Record exact commands and results in the implementation summary.

If tests cannot run because of environment limitations, state:

- command attempted,
- error observed,
- why it appears environmental or task-related,
- what the reviewer should verify.
